# 명세서 탭 리디자인 — 프리미엄 금융 명세서 스타일

## 목적

걱정은행 명세서(진단) 탭의 UI/UX를 전면 개선한다.

- 촌스러운 색상(노란/앰버/올리브)을 제거하고 세련된 금융 문서 톤으로 교체
- 5인 페르소나(또박, 탕감, 철벽, 든든, 불어 + 차곡, 갚어)의 존재감 확대
- CBT 진단의 전문성과 시각적 깊이 강화
- 은행 메타포와 따뜻한 상담 톤 양립

## 범위

- `components/WorryStatement.tsx` — 인트로/퀴즈/결과 3개 phase 전체 리디자인
- `app/page.tsx` — 배경색 등 전역 톤 조정 (명세서 탭 진입 시)
- `components/WorryBankApp.tsx` — 탭 바 스타일 업데이트
- `lib/statement.ts` — 로직 변경 없음, 타입/데이터 그대로 사용

창구 탭(`WorryBank.tsx`)과 `lib/bank.ts`는 이번 범위에서 제외한다.

## 2. 타이포그래피

- **Fredoka**: 로고/헤더 포인트에만 사용
- **Pretendard**: 본문 서체 유지
- 숫자: `tabular-nums` + 약간 큰 사이즈로 재무제표 느낌
- 섹션 제목: `text-sm font-bold tracking-wide uppercase` 스타일로 금융 문서 느낌

## 3. 결과 화면 레이아웃

6개 층으로 정보 위계를 명확히 구분한다.

### 3-1. 1층 — 계좌 요약 헤더

- 딥 네이비 → 슬레이트 그라디언트 배경
- `걱정은행 · 오늘의 명세서` + 오늘 날짜
- 3칸 그리드: 총 걱정 잔고 / 오늘의 이자(골드) / 부실채권(로즈)
- 수치를 `text-2xl font-black`으로 키워 시선 집중

### 3-2. 2층 — 최고 이자 걱정 카드 (핵심 진단)

- 골드(`#d4a574`) 좌측 보더 또는 상단 보더로 "메인 카드" 강조
- 걱정 텍스트, 원금→현재 변화, 배수 표시
- 실현확률 프로그레스 바: 둥근 바 + 퍼센트 라벨
- 하단 태그: 반추 비용, 통제 가능성, 과잉 걱정 지수
- 이자요정 '불어' 코멘트: 카드 하단에 말풍선 스타일 (`bg-slate-50` + 꼬리 삼각형)

### 3-3. 3층 — 직원별 진단 카드 (페르소나 확대)

현재 `Row` 컴포넌트로 나열되던 걱정 자산 분류를 직원별 독립 미니 카드로 분리한다.

각 카드 구조:
```
┌─────────────────────────┐
│ 🐷 적금통 '차곡'        │  ← 직원 이모지 + 이름
│ 미래 불안 적립           │  ← 역할
│                         │
│            150          │  ← 금액 (큰 숫자)
└─────────────────────────┘
```

표시할 직원 카드:
- **차곡** (적금): 미래 불안 적립 금액
- **갚어** (대출): 마음의 빚 금액
- **탕감** (부실채권): 소각 대상 금액 + 건수
- **예금**: 그냥 예치된 걱정 (직원 없이 은행 아이콘)

만기 정보는 이 섹션 하단에 배치.

### 3-4. 보안요원 철벽 경고 (조건부)

- 회피가 감지되었을 때만 표시
- 로즈 좌측 보더 + 화이트 배경 (현재 로즈 배경 → 더 절제된 표현)
- 철벽 이모지 + 경고 메시지

### 3-5. 4층 — 사고 패턴 진단 (CBT 전문성)

- 현재 노란(amber) 배경 제거 → 화이트 카드 + 네이비 좌측 보더라인
- 왜곡 유형 라벨: 네이비 배경 뱃지 (`bg-slate-800 text-white rounded-full px-3 py-1`)
- 정의 텍스트: 본문으로 깔끔하게
- 리프레이밍 질문: 골드 악센트 배경(`bg-amber-50/50`) 별도 박스로 분리, 💬 아이콘

### 3-6. 5층 — 지점장 총평 + 상환 플랜

**든든 총평:**
- 골드 상단 보더 카드
- 지점장 이모지 + 이름 + "총평" 뱃지
- 총평 텍스트

**또박 상환 플랜 (4단계):**
- 세로 타임라인 형태: 좌측에 번호 원(①②③④), 세로 연결선
- 각 단계: 제목(bold) + 상세 설명
- 현재 개별 카드 나열 → 연결된 스텝 시각화

### 3-7. 6층 — 마무리

- 이번 주 상환 완료 뱃지: 에메랄드 톤 유지, 더 미니멀하게
- "다시 진단하기" 버튼: 네이비 배경 (`bg-slate-900`) + 화이트 텍스트
- 면책 문구 유지

## 4. 퀴즈 화면

- **프로그레스**: 상단 바 유지 + 스텝 텍스트 (`1 / 10`)를 네이비 톤으로
- **프로그레스 바 색상**: 현재 slate-950 → 네이비-골드 그라디언트
- **질문 텍스트**: 네이비 컬러, 현재 사이즈 유지
- **선택지 버튼**: 화이트 카드 + 슬레이트 보더, 호버 시 골드 보더 + 미세 lift(`-translate-y-0.5`), 현재 emerald ring 제거
- **이전 질문 버튼**: 현재 스타일 유지 (서브 텍스트)

## 5. 인트로 화면

- 이모지 🎫 유지 (충분히 메타포에 맞음)
- 제목/설명 텍스트: 네이비 톤으로 변경
- CTA 버튼: `bg-slate-900` 네이비 + 화이트 텍스트, 호버 시 `bg-slate-800`
- 카드 배경: `bg-white/90` 유지

## 6. 탭 바 스타일 업데이트

- 현재 베이지(`#ebe2d3`) 배경 → 쿨 그레이(`#e2e8f0`)
- 활성 탭: `bg-white` + 네이비 텍스트 + 미세 shadow
- 비활성 탭: `text-slate-400`

## 7. 변경하지 않는 것

- `lib/statement.ts` 로직 (퀴즈 질문, 점수 계산, 결과 생성 등 전부 유지)
- `lib/bank.ts` (창구 탭 로직)
- `components/WorryBank.tsx` (창구 탭 UI)
- `app/api/teller/route.ts` (API)
- 페르소나 이모지 및 이름 (기존 그대로 사용)
- localStorage 저장 구조

## 성공 기준

1. 명세서 탭 진입 시 "촌스럽다"는 느낌이 사라지고, 세련된 금융 문서 느낌이 든다
2. 5인 직원이 각각 담당 섹션을 "발급"하는 형태로 존재감이 뚜렷하다
3. CBT 진단(사고 패턴)이 시각적으로 전문적이고 읽기 쉽다
4. 은행 메타포와 따뜻한 톤이 유지된다
5. 기존 기능(퀴즈 10문항, 결과 계산)이 그대로 동작한다
